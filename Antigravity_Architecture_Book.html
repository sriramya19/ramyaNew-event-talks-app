Antigravity Architecture

Code Generation & Agentic Loops

"A detailed guide to model interaction, filesystem pipelines, sandboxing, and customizable subagent networks."

Chapter I

The Core Agentic Loop

Antigravity is designed as a collaborative, agentic coding companion. Instead of operating as a single-prompt model, it bridges high-level reasoning with safe, local file and command execution pipelines. It is structured around an autonomous execution circle.

When you present a prompt, the system routes the request through a specialized parser, appending context metrics. The central LLM (like Gemini) constructs a sequential plan and issues tool parameters. These calls are dynamically routed to the local execution sandbox, which contains file readers (view_file, list_dir), file writing editors (write_to_file, replace_file_content), browser monitors, and terminal command processes. The output is processed and returned, continuing the loop until completion.

flowchart TD
    User([User Prompt]) -->|Conversation Context| Model[Gemini Core LLM]
    Model -->|1. Generate Plan & Steps| Loop{Agentic Loop}
    Loop -->|2. Choose Tool Call| Router[Tool Execution Router]
    
    subgraph Sandbox [Execution Sandbox]
        Router -->|Filesystem| FileTools[view_file / write_to_file / replace_file_content]
        Router -->|Shell Commands| Terminal[run_command with User Approval]
        Router -->|Browser / Web| WebTools[read_url_content / search_web]
        Router -->|Sub-Tasks| SubAgent[define_subagent / invoke_subagent]
    end
    
    FileTools -->|Results/Errors| Context[Updated System Context]
    Terminal -->|Terminal Outputs| Context
    WebTools -->|Scraped Content| Context
    SubAgent -->|Reports & Logs| Context
    
    Context -->|Updated Conversation History| Model
    Loop -->|3. Goal Achieved| End([User Response / Clickable Artifacts])

Chapter II

The Four-Step Generation Cycle

Antigravity constructs directories and solves code bugs systematically using an interactive four-phase loop:

  • Context Assembly: Gathers conversational history, files currently active inside the developer's text editor, active cursor placement coordinates, and client platform characteristics.
  • Planning & Tool Selection: The model chooses necessary actions and triggers the relevant tool calls, defining detailed parameter payloads.
  • Execution & Feedback: The local sandbox executes the calls and returns stdout/stderr reports back into the active prompt window.
  • Self-Correction & Linting: If any command errors out or syntax issues are discovered during compilation, the model processes the logs and issues correct edits immediately.

Chapter III

Sandboxing & Safe Command Gates

To secure local filesystems against unintended actions, the execution environment operates under double security gates:

Scope Restrictions: Writing permissions are restricted to the designated active workspace directory. Accessing system file trees, application registries, or external networks requires immediate explicit permission grants.

User Gatekeeping: All shell command proposals (such as launching Flask, installing requirements, or initializing Git repositories) are queued inside the task manager. They are blocked from running until the developer clicks "Approve". No arbitrary shell command runs on your system without your authorization.

Chapter IV

Subagents & Collaborative Workspaces

For large scale engineering tasks, Antigravity delegates complex workflows to specialized subagents:

  • Specialized Profiles: It can spawn a research subagent (read-only) to inspect code patterns or a self subagent (inherits parent configuration) to build helper tools.
  • Workspace Branching: When a subagent is invoked, it can work in an inherited directory, a shared worktree, or a branched isolated workspace (similar to git worktree).
  • High-Concurrency Messaging: Subagents notify the parent agent upon completion, resolving problems in parallel without cluttering your main thread.

Chapter V

Customizations & Plugin System

Antigravity adapts its behavior using a dual-scoped customization system:

  • Global Settings (C:\Users\srira\.gemini\config): Houses universally applied rules, style parameters, and system behaviors.
  • Project Settings (.agents/ / Workspace Root): Manages workspace-specific rules, library constraints, and folder maps.
  • Skills (SKILL.md): Custom task plans defined by the developer. If a task matches a skill's YAML frontmatter title, the instructions are dynamically loaded into the agent's memory.
  • Plugins: Specialized bundles of tool wrappers (such as chrome-devtools-plugin for browser testing and google-antigravity-sdk for SDK access) loaded dynamically based on app profiles.

Chapter VI

Technology Stack & Uses

This project is constructed using a robust stack of programming and scripting languages, each selected for its core strengths:

  • Python (Flask & BeautifulSoup): Acts as the backend scraper and API server, fetching official release notes, caching inputs, and serving structured JSON feeds.
  • JavaScript (ES6+): Drives client interactivity, dynamic timelines, character count validations, localStorage bookmarking, and Mermaid diagram drawing.
  • HTML5 & CSS3: Structures semantic documents and implements responsive glassmorphic layouts, dark/sepia reading themes, and SVG color filters.
  • PowerShell (.NET Speech): Orchestrates local environment initialization, package setups, and synthesizes text paragraphs into high-quality WAV audio books.

Spring AI Release Pulse

Implementation Plan & Architecture

"A roadmap for scraping, parsing, filtering, and publishing Spring AI upgrade notes."

Chapter I

Objectives & Architecture

This document outlines the design and implementation details of the Spring AI Release Pulse web application. The core objective is to dynamically scrape the Spring AI Release Notes and parse updates by release version, grouping changes under category filters.

The system is built on a Flask python backend that serves a JSON API to a vanilla HTML/CSS/JS frontend dashboard. Developers can select updates and compose consolidated tweet summaries directly into an integrated X Web Intent drawer.

graph TD
    A[docs.spring.io] -->|HTTP Scrape| B[Flask app.py]
    B -->|JSON API /api/releases| C[Vanilla HTML/JS Frontend]
    C -->|Interact & Filter| D[Timeline UI]
    C -->|Select Card| E[Tweet Composer Drawer]
    E -->|Copy/Post| F[X / Twitter Web Intent]

Chapter II

Backend Scraping & Parsing

The backend scraping and caching layer is implemented in app.py using Python standard libraries and BeautifulSoup. It requests the official reference upgrade-notes.html using custom User-Agent headers, saving responses in a fast, in-memory cache.

A custom heuristic parsing system extracts and organizes update sections. First, it identifies major version header containers (such as 2.0.0 or 1.1.x). Next, it parses sub-categories like Advisors or Tool Calling. If a sub-category has detailed sub-headings, it generates separate cards for each sub-topic; otherwise, it groups the entire category into a single timeline card. It categorizes updates dynamically based on title keywords matching tags like Advisors, Models, MCP, and Vector Databases.

Chapter III

Frontend Design & CSS

The user interface is designed to wow the user with a modern glassmorphic look. By leveraging native CSS variables, the application transitions smoothly between dark and light themes. Elements are stylized using blur backdrops and subtle border glows matching the Spring Framework brand colors.

The core layout uses a CSS Grid system for displaying metrics and category pills alongside a flexbox-driven timeline list. Active category pills filter updates on-the-fly and display real-time counts, helping developers navigate complex release histories in a simple, readable format.

Chapter IV

UI Logic & X Integration

Interactive client behaviors are managed by app.js. The application dynamically compiles selected cards into the clipboard or the composer drawer. When a user checks one or more cards, a drawer slides in, aggregating titles and descriptions into a ready-to-post draft.

The composer includes a character progress bar that turns red as it approaches the 280-character limit, warning the user of overflow. A one-click button fires a web intent that takes the compiled text directly to X/Twitter, providing a seamless sharing flow for developer advocates.

Spring AI Upgrade Notes

Upgrading to Spring AI 2.0.0

"Key breaking changes, migration steps, and new features in the Spring AI 2.0.0 release."

Chapter I

Advisors & Memory

Upgrading to Spring AI 2.0.0 brings significant renames and updates to the advisor infrastructure. The advisor module has been renamed to align with standard conventions: spring-ai-advisors-vector-store is now spring-ai-vector-store-advisor. Similarly, the ToolSearchToolCallingAdvisor has been moved to its own dedicated module, spring-ai-tool-search-advisor.

Furthermore, chat memory order has been updated to set the default memory precedence order to HIGHEST_PRECEDENCE + 200, which sits outside the default tool-calling order. The ToolCallingAdvisor now manages conversation histories internally. If memory is needed inside the loops, developers must disable the internal history and adjust advisor orders manually.

Chapter II

Tool Calling & Execution

A key breaking change in Spring AI 2.0.0 is the removal of the internalToolExecutionEnabled flag from the options classes. Model-level internal execution loops are gone; instead, tool execution is driven globally by ToolCallingAdvisor within ChatClient, or driven manually via ToolCallingManager when invoking the model directly.

Additionally, the streamToolCallResponses option has been removed from all advisor builders due to design flaws where request streams and response chunks could corrupt downstream memory records. To audit tool calls dynamically, developers should disable auto-registration and manually drive the tool-calling loop.

Chapter III

Options Immutability & Configuration Changes

Options classes, including ChatOptions and EmbeddingOptions, are now strictly immutable. All internal collections are stored as unmodifiable structures. Copy constructors have been removed; modifications must be performed by invoking the new mutate() builder chain pattern instead.

Model configuration properties have also been flattened to remove the .options segment from keys. For example, the property spring.ai.openai.embedding.options.model is now configured directly as spring.ai.openai.embedding.model. Deprecated configurations remain available to guarantee a smooth transition during local builds.

Chapter IV

Databases & Starters

Database schemas have updated to support precise message ordering. The JdbcChatMemoryRepository table adds a new sequence_id column (BIGINT) to guarantee correct ordering across MySQL, Postgres, and SQLite, where timestamp precision alone was non-deterministic. Old databases require schemas to be migrated or rebuilt.

New starters are also introduced to facilitate setup: spring-ai-starter-tool-search-advisor provides auto-configurations for semantic tool filtering. These additions drastically reduce boilerplate configurations and allow search indexing type configurations (regex, lucene, vector) out of the box.

import os
import logging
import urllib.request
from bs4 import BeautifulSoup
from flask import Flask, render_template, jsonify, request

app = Flask(__name__)

# Configure logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

UPGRADE_NOTES_URL = "https://docs.spring.io/spring-ai/reference/upgrade-notes.html"

# Simple in-memory cache
cache = {
    'updates': None,
    'last_updated': None
}

def determine_category(title, parent_category=""):
    t = (title + " " + parent_category).lower()
    if any(word in t for word in ["advisor", "vector-store-advisor"]):
        return "Advisors"
    if any(word in t for word in ["tool", "mcp", "callback", "functioncall"]):
        return "Tool Calling & MCP"
    if any(word in t for word in ["memory", "chatmemory", "history"]):
        return "Chat Memory"
    if any(word in t for word in ["model", "ollama", "openai", "minimax", "genai", "anthropic", "vertex", "bedrock"]):
        return "Models & Providers"
    if any(word in t for word in ["converter", "schema", "output", "structured"]):
        return "Structured Output"
    if any(word in t for word in ["observability", "metric", "span", "trace", "instrument"]):
        return "Observability"
    if any(word in t for word in ["database", "db", "cosmos", "cassandra", "pgvector", "store", "vectordb"]):
        return "Vector Databases"
    if any(word in t for word in ["json", "jackson", "helper"]):
        return "JSON Utilities"
    return "General"

def parse_release_notes(html_content):
    soup = BeautifulSoup(html_content, 'html.parser')
    updates = []
    
    doc = soup.find('article', class_='doc')
    if not doc:
        doc = soup.body
        if not doc:
            return []
            
    sect1_divs = doc.find_all('div', class_='sect1')
    for sect1 in sect1_divs:
        h2 = sect1.find('h2')
        if not h2:
            continue
        version_text = h2.get_text().strip()
        # Clean version text if it contains anchor characters
        version_text = version_text.replace("Upgrading to ", "").strip()
        
        sect2_divs = sect1.find_all('div', class_='sect2')
        for sect2 in sect2_divs:
            h3 = sect2.find('h3')
            if not h3:
                continue
            category_text = h3.get_text().strip()
            category_id = h3.get('id', '')
            
            # Find direct sub-updates (sect3)
            sect3_divs = sect2.find_all('div', class_='sect3')
            
            # Check if all h4 headings in sect3 are generic (Impact, Migration, etc.)
            has_descriptive_subheadings = False
            generic_words = ["impact", "migration", "why", "detail"]
            
            for sect3 in sect3_divs:
                h4 = sect3.find('h4')
                if h4:
                    h4_text = h4.get_text().lower().strip()
                    if not any(gen in h4_text for gen in generic_words):
                        has_descriptive_subheadings = True
                        break
            
            if sect3_divs and has_descriptive_subheadings:
                # Treat each sect3 as an individual update
                for sect3 in sect3_divs:
                    h4 = sect3.find('h4')
                    if not h4:
                        continue
                    update_title = h4.get_text().strip()
                    update_id = h4.get('id', '')
                    
                    # Content is everything inside sect3 except h4
                    content_html = ""
                    for child in sect3.children:
                        if child == h4 or getattr(child, 'name', None) == 'h4':
                            continue
                        content_html += str(child)
                        
                    # Prepend parent category to categories list
                    cat = determine_category(update_title, category_text)
                    
                    updates.append({
                        'id': update_id or f"up-{len(updates)}",
                        'version': version_text,
                        'category': cat,
                        'sub_category': category_text,
                        'title': update_title,
                        'html_content': content_html.strip(),
                        'link': f"{UPGRADE_NOTES_URL}#{update_id}" if update_id else UPGRADE_NOTES_URL
                    })
            else:
                # Treat the entire sect2 as a single update
                content_html = ""
                for child in sect2.children:
                    if child == h3 or getattr(child, 'name', None) == 'h3':
                        continue
                    content_html += str(child)
                    
                cat = determine_category(category_text)
                updates.append({
                    'id': category_id or f"up-{len(updates)}",
                    'version': version_text,
                    'category': cat,
                    'sub_category': 'General',
                    'title': category_text,
                    'html_content': content_html.strip(),
                    'link': f"{UPGRADE_NOTES_URL}#{category_id}" if category_id else UPGRADE_NOTES_URL
                })
                
    return updates

def fetch_and_parse():
    try:
        logger.info(f"Fetching Spring AI Release notes from {UPGRADE_NOTES_URL}")
        req = urllib.request.Request(
            UPGRADE_NOTES_URL, 
            headers={'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36'}
        )
        with urllib.request.urlopen(req, timeout=20) as response:
            html_data = response.read().decode('utf-8')
        
        logger.info("Successfully fetched HTML. Parsing release notes...")
        updates = parse_release_notes(html_data)
        logger.info(f"Successfully parsed {len(updates)} updates.")
        
        cache['updates'] = updates
        import datetime
        cache['last_updated'] = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        return updates, None
    except Exception as e:
        logger.error(f"Error fetching/parsing release notes: {str(e)}")
        return [], str(e)

@app.route('/')
def index():
    return render_template('index.html')

@app.route('/api/releases')
def get_releases():
    force_refresh = request.args.get('force', 'false').lower() == 'true'
    
    if cache['updates'] is not None and not force_refresh:
        logger.info("Returning cached updates")
        return jsonify({
            'status': 'success',
            'releases': cache['updates'],
            'last_updated': cache['last_updated'],
            'cached': True
        })
        
    updates, error = fetch_and_parse()
    if error:
        # If fetch fails but we have cached updates, return them with warning
        if cache['updates'] is not None:
            return jsonify({
                'status': 'warning',
                'releases': cache['updates'],
                'last_updated': cache['last_updated'],
                'message': f"Failed to refresh: {error}. Showing last cached version."
            })
        return jsonify({
            'status': 'error',
            'message': error,
            'releases': []
        }), 500
        
    return jsonify({
        'status': 'success',
        'releases': updates,
        'last_updated': cache['last_updated'],
        'cached': False
    })

if __name__ == '__main__':
    port = int(os.environ.get('PORT', 5000))
    app.run(host='0.0.0.0', port=port, debug=True)

        }
    });
    elements.scrollTopBtn.addEventListener('click', () => {
        window.scrollTo({ top: 0, behavior: 'smooth' });
    });

    // Filter Pills Event Delegation
    elements.filterPills.addEventListener('click', (e) => {
        const pill = e.target.closest('.pill');
        if (!pill) return;
        
        elements.filterPills.querySelectorAll('.pill').forEach(p => p.classList.remove('active'));
        pill.classList.add('active');
        
        state.currentFilter = pill.dataset.filter;

    // ==========================================

    async function fetchUpdates(force = false) {
        showState('loading');
        elements.refreshIcon.classList.add('spinning');
        elements.refreshBtn.disabled = true;

        try {
            const url = force ? '/api/releases?force=true' : '/api/releases';
            const response = await fetch(url);
            if (!response.ok) {
                throw new Error(`Server returned HTTP ${response.status}`);
            }
            const data = await response.json();
            
            if (data.status === 'success' || data.status === 'warning') {
                state.updates = data.releases;
                calculateStats();
                updateFilterPillCounts();
                renderTimeline();
                showState('content');
                
                if (data.status === 'warning') {
                    showToast(data.message, 'error');
                } else {
                    const cacheMsg = data.cached ? 'Loaded cached release notes' : 'Scraped live release notes';
                    showToast(cacheMsg, 'success');
                }
            } else {
                throw new Error(data.message || 'Unknown server error');
            }
        } catch (error) {
            console.error('Error Syncing release notes:', error);
            elements.errorMessageText.textContent = `Scrape Error: ${error.message}`;
            showState('error');
            showToast('Failed to sync release notes', 'error');
        } finally {
            elements.refreshIcon.classList.remove('spinning');
            elements.refreshBtn.disabled = false;
        }

    function calculateStats() {

            elements.valTotalUpdates.textContent = '0';
            elements.valToolingUpdates.textContent = '0';
            elements.valDbUpdates.textContent = '0';
            return;
        }

        // The latest release is the first one parsed
        elements.valLatestVersion.textContent = state.updates[0].version;
        elements.valTotalUpdates.textContent = state.updates.length;

        const tooling = state.updates.filter(u => 
            u.category === 'Advisors' || u.category === 'Tool Calling & MCP'
        ).length;
        
        const dbs = state.updates.filter(u => u.category === 'Vector Databases').length;

        elements.valToolingUpdates.textContent = tooling;
        elements.valDbUpdates.textContent = dbs;
    }

    // ==========================================
    // TIMELINE RENDERER
    // ==========================================

    function renderTimeline() {
        const filtered = filterUpdates();
        
        if (filtered.length === 0) {
            showState('empty');
            return;
        }

        showState('content');
        elements.timelineWrapper.innerHTML = '';

        // Group updates by Version
        const groups = {};
        filtered.forEach(update => {
            if (!groups[update.version]) {
                groups[update.version] = [];
            }
            groups[update.version].push(update);
        });

        // Generate timeline UI elements
        Object.entries(groups).forEach(([version, updates]) => {
            const groupDiv = document.createElement('div');
            groupDiv.className = 'timeline-group';

            // Version marker
            const marker = document.createElement('div');
            marker.className = 'timeline-version-marker';
            marker.innerHTML = `
                <div class="version-dot"></div>
                <div class="version-title">Version ${version}</div>
            `;
            groupDiv.appendChild(marker);

            // Card list container
            const cardsList = document.createElement('div');
            cardsList.className = 'cards-list';

            updates.forEach(update => {
                const card = document.createElement('div');
                const isSelected = state.selectedUpdates.has(update.id);
                card.className = `release-card ${isSelected ? 'selected' : ''}`;
                card.dataset.id = update.id;

                const badgeClass = getBadgeClass(update.category);

                card.innerHTML = `

                        <div class="custom-checkbox" aria-label="Select update to tweet">

                    </div>
                    <div class="card-body">
                        <div class="card-header">
                            <div class="card-meta">
                                <span class="badge ${badgeClass}">${update.category}</span>
                                <span class="sub-category-tag">${update.sub_category}</span>
                            </div>
                            <div class="card-actions">
                                <button class="btn-inline-action copy-action" title="Copy update text">

                                    <i class="fa-brands fa-x-twitter"></i>

                            </div>

                        <h3 class="card-title">${update.title}</h3>
                        <div class="card-text">
                            ${update.html_content}
                        </div>
                    </div>
                `;

                // Card Click - toggle selection (prevent triggers on interactive elements)
                card.addEventListener('click', (e) => {
                    if (e.target.closest('a') || e.target.closest('.btn-inline-action') || e.target.closest('pre') || e.target.closest('code')) {
                        return;

                    const cleanText = `${update.title}\n\n${cleanHtml(update.html_content)}`;

    function filterUpdates() {
        return state.updates.filter(update => {
            // Pill filter match
            if (state.currentFilter !== 'all' && update.category !== state.currentFilter) {
                return false;
            }
            
            // Search query match
            if (state.searchQuery.trim() !== '') {
                const query = state.searchQuery.toLowerCase();
                const inTitle = update.title.toLowerCase().includes(query);
                const inContent = update.html_content.toLowerCase().includes(query);
                const inCategory = update.category.toLowerCase().includes(query);
                const inSubCategory = update.sub_category.toLowerCase().includes(query);
                const inVersion = update.version.toLowerCase().includes(query);
                
                return inTitle || inContent || inCategory || inSubCategory || inVersion;
            }

            return true;
        });
    }

    function updateTweetComposer() {
        const count = state.selectedUpdates.size;
        elements.selectedBadge.textContent = `${count} selected`;

        if (count === 0) {
            toggleDrawer(false);
            return;
        }

        toggleDrawer(true);

        const selectedItems = state.updates.filter(u => state.selectedUpdates.has(u.id));
        let composedText = '';

        if (selectedItems.length === 1) {
            composedText = formatSingleTweet(selectedItems[0]);
        } else {
            // Thread/Bullet list consolidation
            const version = selectedItems[0].version;
            composedText = `Spring AI Release Updates (v${version}):\n`;
            
            selectedItems.forEach((u, idx) => {
                const bullet = `\n${idx + 1}. [${u.category}] ${u.title}`;
                composedText += bullet;
            });
            
            // Add a link footer
            composedText += `\n\nRead the full details at docs.spring.io/spring-ai`;
        }

        elements.tweetTextarea.value = composedText;
        updateCharProgressBar();
    }

    function formatSingleTweet(update) {
        const cleanContent = cleanHtml(update.html_content);
        // Truncate cleanly for single tweet to keep under X limit if content is too long
        let text = cleanContent;
        if (text.length > 180) {
            text = text.substring(0, 177) + '...';
        }
        return `Spring AI Update (v${update.version}):\n\n[${update.category}] ${update.title}\n${text}\n\nDocs: ${update.link}`;
    }

    function updateCharProgressBar() {
        const text = elements.tweetTextarea.value;
        const count = text.length;
        elements.charCountText.textContent = `${count} / 280`;

        const percent = Math.min((count / 280) * 100, 100);
        elements.progressFill.style.width = `${percent}%`;

        // Update colors depending on remaining character threshold
        const container = elements.charCountText.parentNode;
        container.className = 'char-details';
        elements.progressFill.style.backgroundColor = 'var(--primary)';

        if (count > 280) {
            container.classList.add('danger');
            elements.progressFill.style.backgroundColor = 'var(--accent-red)';
        } else if (count > 240) {
            container.classList.add('warning');
            elements.progressFill.style.backgroundColor = 'var(--accent-orange)';
        }
    }

        const el = document.createElement('textarea');
        el.value = text;

        el.select();
        document.execCommand('copy');

    function cleanHtml(htmlStr) {
        const temp = document.createElement('div');
        temp.innerHTML = htmlStr;
        
        // Strip out pre blocks/code blocks from text to keep tweet concise
        temp.querySelectorAll('.listingblock, pre').forEach(el => el.remove());
        
        // Replace inner codes with backticks to look readable on Twitter
        temp.querySelectorAll('code').forEach(c => {
            c.replaceWith(`\`${c.textContent}\``);
        });

        let text = temp.textContent || temp.innerText || "";
        // Replace consecutive newlines or whitespaces
        return text.replace(/\s+/g, ' ').trim();
    }

    function showState(mode) {
        elements.loadingState.classList.add('hidden');
        elements.errorState.classList.add('hidden');
        elements.emptyState.classList.add('hidden');
        elements.timelineWrapper.classList.add('hidden');

        if (mode === 'loading') elements.loadingState.classList.remove('hidden');
        else if (mode === 'error') elements.errorState.classList.remove('hidden');
        else if (mode === 'empty') elements.emptyState.classList.remove('hidden');
        else if (mode === 'content') elements.timelineWrapper.classList.remove('hidden');
    }

    function showToast(message, type = 'info') {
        const toast = document.createElement('div');
        toast.className = `toast ${type}`;
        
        let icon = 'fa-circle-info';
        if (type === 'success') icon = 'fa-circle-check';
        if (type === 'error') icon = 'fa-triangle-exclamation';

        toast.innerHTML = `
            <i class="fa-solid ${icon}"></i>
            <span>${message}</span>
            <button class="toast-close" aria-label="Close alert"><i class="fa-solid fa-xmark"></i></button>
        `;
        
        // Bind click on close button
        toast.querySelector('.toast-close').addEventListener('click', (e) => {
            e.stopPropagation();
            toast.remove();
        });

        elements.toastContainer.appendChild(toast);
        
        // Exit animation timer
        setTimeout(() => {
            if (toast.parentNode) {
                toast.style.transition = 'opacity 0.4s ease, transform 0.4s ease';
                toast.style.opacity = '0';
                toast.style.transform = 'translateX(50px)';
                setTimeout(() => toast.remove(), 400);
            }
        }, 4000);
    }

        };

        elements.filterPills.querySelectorAll('.pill').forEach(pill => {
            const filter = pill.dataset.filter;

            let label = filter;

    function exportToCSV() {
        const filtered = filterUpdates();
        if (filtered.length === 0) {
            showToast('No updates to export', 'error');
            return;
        }

        const headers = ['Version', 'Category', 'Sub-Category', 'Title', 'Description', 'Link'];
        
        const rows = filtered.map(u => [
            u.version,
            u.category,
            u.sub_category,
            u.title,
            cleanHtml(u.html_content),
            u.link
        ]);

        const csvContent = [
            headers.join(','),
            ...rows.map(row => row.map(val => `"${String(val).replace(/"/g, '""')}"`).join(','))
        ].join('\n');

        const blob = new Blob([csvContent], { type: 'text/csv;charset=utf-8;' });
        const link = document.createElement('a');
        const url = URL.createObjectURL(blob);
        
        link.setAttribute('href', url);
        link.setAttribute('download', `spring_ai_release_notes.csv`);
        link.style.visibility = 'hidden';
        document.body.appendChild(link);
        link.click();
        document.body.removeChild(link);

        showToast(`Exported ${filtered.length} updates to CSV`, 'success');
    }

    function initTheme() {
        if (state.theme === 'light') {
            elements.body.classList.remove('dark-theme');
            elements.body.classList.add('light-theme');
            elements.themeToggle.innerHTML = '<i class="fa-solid fa-sun"></i>';

            elements.body.classList.remove('light-theme');
            elements.themeToggle.innerHTML = '<i class="fa-solid fa-moon"></i>';
        }
    }

    function toggleTheme() {
        if (elements.body.classList.contains('dark-theme')) {
            elements.body.classList.remove('dark-theme');
            elements.body.classList.add('light-theme');
            elements.themeToggle.innerHTML = '<i class="fa-solid fa-sun"></i>';
            state.theme = 'light';
        } else {
            elements.body.classList.add('dark-theme');
            elements.body.classList.remove('light-theme');
            elements.themeToggle.innerHTML = '<i class="fa-solid fa-moon"></i>';
            state.theme = 'dark';
        }
        localStorage.setItem('theme', state.theme);
        showToast(`Theme switched to ${state.theme} mode`, 'info');
    }
